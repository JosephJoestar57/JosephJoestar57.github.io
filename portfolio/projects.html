Terraria King Slime Sprite Project (CS1)

This project that I created in CS1 was very unpredictable. In this
project, I used pillow coding on python "which we learned about in class"
to create create pixel art gif animations of a terraria boss's sprites that
would be in the actual game. What I didn't expect though, was just how time
consuming it would be...
I think that a large part of the time consuming nature of the project was due
to the code that I used to create it. I feel like my optimization of the code
was underwhelming, and due to that it turned out to take way longer than it
should have...
What I believe that I learned from this project was that I should focus on
creating the most optimal code for something to make it more efficient, then
begin to actually start the project. My biggest flaw with it was that I only
took around one week of the project time to optimize the code, then I focused
on creating the detail within the artistic aspect of it. I think that if I
had just spent another week or two making the code more efficient, than I
could have had twice as much of a result...
In conclusion to my project though, I was at the very least able to produce
my minimum expectations for it. I still was able to create all of the animations
and an extra hit animation that would be displayed in the actual boss fight
in terraria, so I still was able to create a result, but I am disapointed that
I couldn't create another boss's sprite animations...
It was a learning experience though, and at least now I know what I need to do for
the next time that I have a project during my time taking CS as a subject
